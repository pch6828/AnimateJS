# Codex 프롬프트 세트
## Retro Newspaper A–Z Typography Personal Page

이 문서는 `personal-page-dev-plan.md`를 바탕으로, **Codex에 순차적으로 입력하기 좋은 프롬프트 세트**로 재구성한 문서다.  
목표는 기존 **Vite + React 기반 프로젝트를 리팩토링**하여, 레트로 신문 감성의 **A~Z 인터랙티브 개인 페이지**를 완성하는 것이다.

---

## 1. 사용 방식

이 문서는 아래 3가지 방식으로 사용할 수 있다.

### 방식 A. 순차 실행
가장 추천하는 방식이다.

1. `공통 컨텍스트 프롬프트`를 먼저 입력한다.
2. 그다음 `0. 프로젝트 감사`부터 순서대로 하나씩 입력한다.
3. 각 단계 결과를 반영한 뒤 다음 프롬프트로 넘어간다.

### 방식 B. 필요한 작업만 선택 실행
이미 일부 구현이 되어 있다면, 해당 단계 프롬프트만 골라 사용한다.

### 방식 C. 한 번에 큰 방향 정리
코드를 바로 수정하기 전에 `마스터 프롬프트`와 `프로젝트 감사 프롬프트`만 먼저 실행해, 전체 구조 제안을 받는다.

---

## 2. 먼저 붙여넣을 공통 컨텍스트 프롬프트

아래 프롬프트는 세션 초반에 한 번 넣어 두면 좋다.

```md
너는 Vite + React 기반 프론트엔드 리팩토링을 수행하는 시니어 웹 개발자다.

내 프로젝트 목표는 다음과 같다.
- 개인 페이지를 리팩토링한다.
- 전체 컨셉은 레트로 신문 감성이다.
- 메인 폰트는 Times New Roman 계열의 세리프 중심이다.
- 메인페이지에는 A~Z 이니셜이 원형으로 배치된 다이얼이 있고, 마우스/터치로 회전 가능해야 한다.
- 이니셜 클릭 시 세부 페이지로 이동한다.
- 세부 페이지에는 HTML5 canvas 기반 인터랙티브 타이포그래피 영역과 설명 패널이 함께 있다.
- A~Z 전체는 데이터 중심 구조로 관리한다.
- Vite + React를 사용한다.
- GitHub Pages 배포를 고려한다.
- 정적 호스팅 안정성을 위해 우선 HashRouter를 사용한다.

구현 원칙은 다음과 같다.
- 메인 다이얼은 Canvas가 아니라 DOM + CSS transform + pointer event 중심으로 구현한다.
- 세부 페이지의 작품 영역만 Canvas를 사용한다.
- 기존 프로젝트를 최대한 리팩토링하는 방향으로 진행한다.
- 불필요한 라이브러리는 추가하지 않는다.
- 과한 시각 효과보다 타이포그래피, 여백, 구분선, 편집 구조를 우선한다.
- 접근성, 반응형, 성능을 기본적으로 고려한다.
- 모바일에서도 자연스럽게 동작해야 한다.
- 설명 텍스트가 Canvas의 의미를 보완해야 한다.

작업 방식은 다음과 같다.
- 먼저 현재 프로젝트 구조를 파악하고, 수정 전략을 짧게 설명한 뒤 구현한다.
- 파일 단위로 무엇을 만들고 수정하는지 명확히 설명한다.
- 가능한 경우 전체 diff 또는 파일별 최종 코드를 제안한다.
- 내가 바로 반영할 수 있도록 실행 가능한 코드 중심으로 답한다.
- 추상적 설명보다 실제 구현을 우선한다.
```

## 3. Codex에게 항상 같이 주면 좋은 작업 규칙

필요할 때 아래 규칙도 함께 붙여 넣는다.

```md
다음 규칙을 지켜서 작업해줘.

1. 기존 코드베이스를 먼저 읽고, 재사용 가능한 부분과 제거할 부분을 구분해줘.
2. 큰 구조 변경이 필요하면 먼저 제안하고, 그다음 실제 수정안을 작성해줘.
3. 가능한 한 적은 파일 변경으로 시작하되, 구조적으로 필요한 분리는 해줘.
4. React 컴포넌트는 역할별로 분리하고, props 이름은 의미가 분명하게 작성해줘.
5. pointer/mouse/touch 이벤트는 중복 구현하지 말고 pointer event 중심으로 정리해줘.
6. Canvas 관련 코드는 공통 엔진과 개별 sketch 모듈로 분리해줘.
7. 라우팅은 HashRouter 기준으로 구현해줘.
8. GitHub Pages 배포를 고려해 경로와 base 설정을 안전하게 처리해줘.
9. 스타일은 레트로 신문 감성이 나도록 하되, 과한 빈티지 효과는 피하고 읽기 쉬운 구조를 우선해줘.
10. 답변 끝에는 변경 파일 목록, 핵심 구현 포인트, 다음 작업 추천을 함께 적어줘.
```

## 4. 마스터 프롬프트

프로젝트 전체 방향을 한 번에 정리받고 싶을 때 사용한다.

```md
현재 내 프로젝트는 Vite + React 기반 개인 페이지이며, 기존 구현을 리팩토링하려고 한다.

목표:
- 레트로 신문 감성의 개인 페이지
- 메인페이지에 A~Z 원형 다이얼
- 마우스/터치로 다이얼 회전
- 이니셜 클릭 시 상세 페이지 이동
- 세부 페이지에 Canvas 기반 인터랙티브 타이포그래피 + 설명 패널
- A~Z는 데이터 중심 구조로 관리
- GitHub Pages 배포 예정
- HashRouter 우선 사용

원하는 산출물:
1. 현재 프로젝트를 어떤 구조로 리팩토링하면 좋을지 제안
2. 추천 폴더 구조
3. 단계별 구현 순서
4. 각 단계에서 수정해야 할 파일 목록
5. 바로 착수 가능한 최소 구현안

주의:
- 메인 다이얼은 DOM 기반으로 구현해줘.
- 세부 타이포그래피 영역만 Canvas를 사용해줘.
- 스타일은 Times New Roman 계열, 종이색 배경, 잉크색 텍스트, 얇은 구분선 중심으로 잡아줘.
- 가능하면 기존 코드를 재사용하는 방향으로 제안해줘.
- 답변은 추상적 설명보다 실제 리팩토링 계획과 코드 구조 중심으로 해줘.
```

## 5. 단계별 프롬프트

## 5-0. 프로젝트 감사 / 리팩토링 진단

기존 프로젝트를 먼저 읽게 하고 싶을 때 사용한다.

```md
현재 프로젝트를 먼저 분석해서 리팩토링 계획을 세워줘.

내 목표는 레트로 신문 감성의 A~Z 인터랙티브 개인 페이지로 리팩토링하는 것이다.
기술 스택은 Vite + React이고 GitHub Pages에 배포할 예정이다.

다음 형식으로 답해줘.
1. 현재 구조 요약
2. 재사용 가능한 컴포넌트/스타일/유틸
3. 제거하거나 교체해야 할 부분
4. 우선순위가 높은 기술 부채
5. 최소 변경으로 시작하는 리팩토링 순서
6. 권장 폴더 구조
7. 첫 번째 실제 수정안

가능하면 실제 파일명을 기준으로 설명해줘.
```

## 5-1. 디자인 토큰 및 레이아웃 셸 구축

```md
이 프로젝트에 맞는 기본 디자인 시스템과 레이아웃 셸을 만들어줘.

- 레트로 신문 감성

- 종이색 배경, 잉크색 텍스트, 얇은 구분선

- 현대적인 카드 UI보다 편집물 같은 레이아웃

답변 형식:
1. 생성/수정 파일 목록
2. 각 파일의 최종 코드
```

```md
React로 A~Z 원형 다이얼 컴포넌트를 구현해줘.

요구사항:

- DOM + CSS transform 기반 구현

- 키보드 포커스 스타일도 고려

- `LetterDial`
- 필요하면 `LetterDialItem`
- `useDialRotation` 훅 분리 가능

원하는 결과:
1. 컴포넌트 구조 제안
2. 필요한 파일 목록
3. 각 파일의 최종 코드

5. 모바일 대응 포인트
```

## 11. 마지막 팁

Codex에는 한 번에 너무 많은 요구를 넣기보다, 아래처럼 요청하는 편이 좋다.

- 먼저 구조 제안
- 그다음 실제 파일 코드 생성
- 그다음 스타일 미세 조정
- 그다음 인터랙션 개선
- 그다음 성능/접근성/배포 점검

특히 이 프로젝트는 **다이얼**, **상세 레이아웃**, **Canvas 엔진**, **개별 sketch**가 서로 성격이 다르므로, 한 번에 한 축씩 구현시키는 것이 가장 안정적이다.

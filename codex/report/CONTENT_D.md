# D - Designer 보고서

## 1. 작업 개요
- 대상 파일: `src/animate-js/class/animate/d_designer.jsx`
- 테마: 전환
- 목표: `Designer` 타이포그래피가 드래그에 반응해 `Developer`로 전환되고, 반대 방향 드래그로 다시 돌아오는 인터랙션 구현

## 2. 구현 내용
- 기존의 드로잉 경로 기반 인터랙션을 제거하고, 드래그 누적값으로 전환 상태를 제어하는 구조로 전면 교체했습니다.
- `Designer`와 `Developer`를 각각 오프스크린 캔버스에 렌더링한 뒤, 화면을 세로 슬라이스로 분할해 각 열이 독립적으로 회전하는 것처럼 보이도록 구성했습니다.
- 전환은 한 번에 바뀌지 않고, 왼쪽 열부터 순차적으로 뒤집히도록 스태거를 주어 "세로 열이 돌아가며 다른 면이 나오는" 감각을 만들었습니다.
- 슬라이스마다 폭 축소, 그림자, 하이라이트, 바닥 그림자를 함께 적용해 평면 교체가 아니라 회전하는 면처럼 읽히도록 보정했습니다.
- 현재 상태가 어느 쪽에 가까운지 알 수 있도록 하단에 `Designer`, `Developer` 라벨의 강조도도 함께 변화하도록 넣었습니다.

## 3. 인터랙션 설계
- 초기 상태는 `Designer`입니다.
- 마우스를 누른 채 오른쪽으로 드래그하면 전환 진행도가 증가합니다.
- 마우스를 누른 채 왼쪽으로 드래그하면 전환 진행도가 감소합니다.
- 드래그를 멈춘 뒤에는 마지막 상태를 유지하고, 화면은 부드럽게 해당 진행도에 수렴합니다.

## 4. 시각 설계 포인트
- 종이 질감에 가까운 따뜻한 배경과 프레임을 사용해 기존 프로젝트의 레트로한 분위기와 어긋나지 않도록 맞췄습니다.
- 전면 `Designer`는 브라운 계열, 후면 `Developer`는 블루 계열로 구분해 전환 결과가 즉시 인지되도록 했습니다.
- 슬라이스 회전 중에는 셰이딩과 좁아지는 폭을 이용해 3D 플립에 가까운 인상을 주도록 처리했습니다.

## 5. 정리
- 이번 작업으로 `D` 페이지는 기존 "긁어서 드러내기" 방식에서, 테마에 직접 연결되는 "역할의 전환" 인터랙션으로 바뀌었습니다.
- 구현은 기존 콘텐츠 페이지 구조를 유지한 채 `AnimationD`, `CleanD`, 설명/툴팁 데이터만 교체하는 방식으로 마무리했습니다.

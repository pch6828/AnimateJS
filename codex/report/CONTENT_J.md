# CONTENT J 개발 보고서

## 개요
- 파일: `src/animate-js/class/animate/j_judging.jsx`
- 주제: `mbti-J`를 계획표 테마의 클립보드 인터랙션으로 재구성

## 작업 목표
- 기존 `J` 애니메이션을 계획표 중심의 새로운 콘셉트로 다시 개발
- 클립보드 위에 `Plan J`가 적힌 종이가 보이도록 구성
- 사용자가 종이를 마우스로 잡아 끌면 종이가 떨어져 나가도록 구현
- 종이가 떨어진 뒤 아래에 있던 다음 계획표가 자연스럽게 나타나도록 구성
- 종이에 적히는 알파벳은 `A`부터 `Z`까지 순환하고, `Z` 다음에는 다시 `A`로 돌아오도록 구현

## 최종 구현 요약
1. `J` 애니메이션을 전면 재구성했습니다.
- 기존의 나무/씨앗 기반 로직을 제거하고, 캔버스 위에 `클립보드 + 종이 스택`을 그리는 구조로 교체했습니다.
- 메인 오브젝트는 상단의 현재 종이 1장과 그 아래에 겹쳐 있는 다음 종이들로 구성했습니다.

2. 계획표 콘셉트에 맞는 시각 구성을 추가했습니다.
- 배경은 따뜻한 베이지 계열 그라디언트로 정리하고, 중앙에는 나무색 클립보드를 배치했습니다.
- 종이는 줄노트 형태의 선과 여백선을 넣어 실제 계획표처럼 보이도록 구성했습니다.
- 상단 종이에는 `Plan <letter>` 형식의 제목이 보이도록 만들었습니다.

3. 드래그 인터랙션을 상태 기반으로 정리했습니다.
- 포인터가 종이 영역 안에서 눌렸을 때만 드래그가 시작되도록 처리했습니다.
- 드래그 중에는 종이의 위치와 회전이 함께 변하도록 해 손으로 잡아당기는 느낌을 넣었습니다.
- 충분히 아래로 끌어내리면 종이가 클립보드에서 떨어진 것으로 판정되도록 구성했습니다.

4. 종이 낙하와 다음 종이 등장 흐름을 구현했습니다.
- 종이가 분리되면 속도와 중력을 적용해 화면 아래로 떨어지도록 만들었습니다.
- 현재 종이가 화면 밖으로 사라지면 내부 인덱스를 다음 알파벳으로 갱신하고, 새 종이를 상단에 다시 생성하도록 처리했습니다.
- 아래에 깔린 종이들은 약간씩 어긋난 위치와 각도로 배치해서 다음 계획표가 대기 중인 느낌을 주도록 했습니다.

5. 알파벳 순환 로직을 추가했습니다.
- 현재 종이의 시작값은 요청에 맞춰 `J`로 설정했습니다.
- 종이가 한 장 떨어질 때마다 `K`, `L`, `M` 순서로 이어지며, `Z`에 도달하면 다시 `A`부터 시작하도록 구현했습니다.

6. 설명 텍스트와 툴팁도 새 콘셉트에 맞게 정리했습니다.
- `descriptionJ`에는 계획을 여러 장으로 준비하는 J 성향의 이미지와 이번 인터랙션 의도를 반영했습니다.
- `toolTipJ`에는 종이를 끌어내리는 사용 방법과 알파벳 순환 동작을 안내하도록 내용을 갱신했습니다.

## 주요 로직
- 상태 객체에 현재 알파벳 인덱스, 드래그 여부, 종이 위치, 낙하 속도, 회전값을 함께 저장합니다.
- 종이를 클릭한 뒤 드래그하면 현재 종이의 오프셋과 회전값이 실시간으로 갱신됩니다.
- 드래그 거리가 기준치를 넘으면 종이가 분리되고, 이후에는 중력 기반 낙하 애니메이션으로 전환됩니다.
- 종이가 화면 아래로 완전히 벗어나면 다음 알파벳 인덱스를 계산하고 새 종이 상태를 초기화합니다.
- 알파벳은 `ABCDEFGHIJKLMNOPQRSTUVWXYZ` 문자열을 기준으로 순환 계산해 `Z -> A` 흐름을 유지합니다.

## 결과
- `mbti-J`를 상징하는 “계획표를 여러 장 준비해 두는 사람”이라는 이미지가 이전보다 직접적으로 전달되도록 바뀌었습니다.
- 사용자가 종이를 직접 끌어내리는 행동과 다음 계획표가 바로 이어서 나타나는 흐름이 자연스럽게 연결되도록 정리했습니다.
- 단순히 텍스트를 보여주는 화면이 아니라, 계획이 계속 이어지는 성향을 인터랙션으로 표현하는 방향으로 개선했습니다.

## 검증
- `npm run build`로 번들 단계 검증을 진행했습니다.
- 빌드가 정상 완료되면 이번 구현은 문법 및 번들 기준으로 문제없는 상태입니다.

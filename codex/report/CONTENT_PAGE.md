# CONTENT_PAGE 작업 보고서

## 작업 개요
- 대상 파일: `src/animate-js/class/content_page/content.jsx`, `src/animate-js/style/content.css`
- 보고 목적: 콘텐츠 페이지를 프로젝트 전체 컨셉과 맞는 레트로 신문형 디자인 시스템과 레이아웃 셸로 정비
- 기준 문서: `codex/BASE_PROMPT.md`

## 반영한 방향
- Times New Roman 계열 세리프 중심으로 타이포그래피를 재구성했습니다.
- 종이색 배경, 잉크색 텍스트, 얇은 구분선을 기준으로 콘텐츠 페이지 전용 토큰을 정리했습니다.
- 기존의 단순 좌우 분할 화면을 마스트헤드, 기사 패널, 인터랙티브 플레이트 구조로 바꿨습니다.
- 현대적인 카드 UI 대신 인쇄물 편집면처럼 보이도록 여백과 선 중심의 구조를 사용했습니다.
- CSS는 페이지 범위 내부 변수와 명확한 클래스 역할 중심으로 정리해 유지보수성을 높였습니다.

## 상세 변경 사항

### 1. `content.jsx`
- 상단에 마스트헤드와 호수 메타 정보를 추가했습니다.
- 좌측 설명 영역을 기사형 패널로 재구성했습니다.
- 제목, 이니셜, 도입 문장, 본문 단락을 분리해 읽기 흐름을 정리했습니다.
- 기존 `?` 버튼 UI를 `Show notes / Hide notes` 버튼으로 교체해 의미를 분명히 했습니다.
- 툴팁을 말풍선 스타일 대신 편집 노트 스타일 오버레이로 변경했습니다.
- 캔버스 인터랙션 상태를 `useRef`로 관리하도록 바꿔서, 포인터 이동 때마다 애니메이션 effect가 다시 생성되지 않도록 정리했습니다.
- 리사이즈 이벤트 정리와 `requestAnimationFrame` 정리를 effect 내부에서 함께 처리했습니다.

### 2. `content.css`
- 콘텐츠 페이지 전용 색상/선/배경 변수를 정의했습니다.
- 전체 페이지를 `masthead + two-column layout` 구조로 구성했습니다.
- 좌측은 기사 지면, 우측은 인터랙티브 지면으로 보이도록 선과 여백을 재정렬했습니다.
- 제목과 본문은 세리프 중심으로, 메타 정보는 작은 대문자 스타일로 맞췄습니다.
- 캔버스 영역은 종이 위 삽화판처럼 보이도록 프레임을 주고 배경 톤을 분리했습니다.
- 모바일 비율에서는 단일 컬럼으로 자연스럽게 내려오도록 반응형 규칙을 유지했습니다.

## 기대 효과
- 메인 페이지의 신문 감성과 콘텐츠 페이지의 분위기가 더 자연스럽게 연결됩니다.
- 설명 텍스트가 단순 부가 정보가 아니라 기사 본문처럼 읽히게 됩니다.
- 인터랙티브 캔버스는 유지하면서도 페이지 전체 인상이 더 편집물답게 정리됩니다.
- 추후 다른 상세 페이지에도 같은 토큰과 구조를 재사용하기 쉬워집니다.

## 확인 사항
- 별도 테스트 명령은 실행하지 않았습니다.
- 기존 애니메이션 모듈의 동작 방식은 유지하고, 페이지 셸과 인터랙션 연결 방식만 정리했습니다.

## 추가 수정 사항
- 사용자 피드백에 맞춰 화면 요소를 `캔버스`와 `설명란` 두 영역 중심으로 다시 단순화했습니다.
- 기존의 마스트헤드, 별도 스테이지 헤더, 캔버스 오버레이 노트는 제거했습니다.
- 페이지 진입 시 애니메이션이 바로 동작하지 않던 문제는 `useParams()`로 받은 `id`를 대문자로 정규화한 뒤 `items`에서 조회하도록 수정해 해결했습니다.
- 인터랙션 안내는 별도 오버레이 대신 설명란 내부 토글 섹션으로 옮겨 화면 복잡도를 낮췄습니다.

## 후속 수정 사항
- 데스크톱 배치를 `설명란 좌측 / 캔버스 우측`으로 재조정했습니다.
- 전환 효과가 진입 시 빠지던 문제는 `CSSTransition`이 실제 DOM 래퍼에 클래스가 붙도록 구조를 바꾸고, `className` 대신 `classNames`를 사용해 수정했습니다.
- 전환 스타일은 페이지 래퍼 기준으로 다시 작성해 콘텐츠 페이지가 진입과 이탈 모두에서 같은 방향성으로 움직이도록 정리했습니다.

## 디테일 조정
- 설명란과 캔버스 비율을 약 `3.5 : 6.5`에 가깝게 조정했습니다.
- 콘텐츠 페이지 전환 시 설명란과 캔버스가 같은 방향에서 함께 진입하고 함께 이탈하도록 애니메이션 방향을 통일했습니다.
- 설명란과 캔버스의 시작 위치도 뷰포트 기준으로 맞춰, 두 영역이 같은 화면 밖 지점에서 동시에 들어오는 인상으로 보이도록 조정했습니다.
- 콘텐츠 페이지 래퍼의 전체 배경은 제거하고 설명란 자체에 종이 배경을 주어, 전환 완료 전까지는 메인 페이지가 뒤에 자연스럽게 남아 보이도록 정리했습니다.

## 반응형 1차 작업
- 메인 페이지와 동일하게 `가로:세로 = 6:5`를 기준으로 콘텐츠 페이지의 compact 상태를 판정하도록 맞췄습니다.
- compact 상태에서는 설명란을 기본으로 숨기고, 토글 버튼으로 열고 닫을 수 있게 변경했습니다.
- compact 상태의 설명란은 전체 레이아웃을 아래로 밀지 않고, 캔버스 위에서 열리는 좌측 드로어 형태로 구성했습니다.
- compact 상태에서는 `Back to Index` 버튼을 설명란이 아니라 캔버스 상단에 배치해, 설명란이 닫혀 있어도 바로 이동할 수 있게 했습니다.
- compact 상태의 페이지 전환은 좌우 이동 대신 `아래에서 위`로 바꾸고, 설명란은 움직이지 않은 채 캔버스만 전환되도록 조정했습니다.
- 추가로 compact 상태의 전환 중에는 설명란이 전혀 보이지 않도록 숨김 처리했습니다.

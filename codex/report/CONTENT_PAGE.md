## 최종 반영 결과

### 1. 레이아웃 구조 정리
- 콘텐츠 페이지 구조를 `설명란 + 캔버스` 중심으로 단순화했습니다.
- 불필요한 마스트헤드, 별도 스테이지 헤더, 과한 오버레이 요소는 제거했습니다.
- 현재 데스크톱에서는 설명란과 캔버스가 좌우 배치되며, 비율은 약 `3.5 : 6.5`로 맞췄습니다.
- 설명란이 길어질 경우 잘리지 않도록 내부 스크롤이 가능하게 했습니다.
- 스크롤은 유지하되 스크롤바는 보이지 않도록 숨김 처리했습니다.

### 2. 시각 스타일 정리
- Times New Roman 계열 세리프 중심의 타이포그래피로 정리했습니다.
- 종이색 배경, 잉크색 텍스트, 얇은 구분선 중심으로 색과 분위기를 맞췄습니다.
- 넓은 화면에서는 캔버스에 그림자를 두고, 좁은 화면에서는 설명란 드로어에 그림자를 두도록 분리했습니다.
- 그림자가 더 잘 보이도록 강도도 추가 조정했습니다.

### 3. 애니메이션/로직 안정화
- 라우트 파라미터 `id`를 대문자로 정규화한 뒤 데이터 `Map`에서 조회하도록 수정했습니다.
- 이 변경으로 content page 진입 시 애니메이션이 바로 정상 동작하도록 정리했습니다.
- 포인터 인터랙션 상태는 `useRef`로 관리해 불필요한 re-render와 effect 재생성을 줄였습니다.
- 캔버스 리사이즈와 `requestAnimationFrame` 정리를 effect 내부에서 함께 처리했습니다.

### 4. 페이지 전환 효과 정리
- `CSSTransition`이 실제 DOM 래퍼에 클래스가 붙도록 전환 구조를 정리했습니다.
- content page 진입/이탈 모두 전환 효과가 정상 동작하도록 수정했습니다.
- 전환 중 메인 페이지가 갑자기 사라지지 않도록, content page 전체 배경이 먼저 화면을 덮지 않게 조정했습니다.
- 설명란과 캔버스의 시작 위치를 맞춰 같은 화면 바깥 지점에서 들어오는 인상으로 정리했습니다.

### 5. 반응형 작업
- 메인 페이지와 동일하게 `가로:세로 = 6:5`를 compact 기준으로 사용했습니다.
- compact 상태에서는 설명란을 기본으로 숨기고, 버튼으로 열고 닫을 수 있게 했습니다.
- 좁은 화면에서 설명란은 레이아웃을 미는 방식이 아니라 좌측 드로어처럼 열리도록 구성했습니다.
- 좁은 화면에서는 `Back to Index` 버튼을 설명란이 아니라 캔버스 상단에 배치했습니다.
- `Show Description` 버튼도 캔버스 쪽 컨트롤 그룹으로 옮겨 두 버튼이 함께 움직이도록 맞췄습니다.
- 두 버튼이 붙어 보이지 않도록 간격도 조정했습니다.

### 6. 좁은 화면 전환 상세 조정
- 좁은 화면에서는 페이지 전환 방향을 `아래에서 위`로 변경했습니다.
- 이때 설명란은 전환에 참여하지 않고, 캔버스만 이동하도록 분리했습니다.
- 전환 중에는 설명란이 아예 보이지 않도록 숨김 처리했습니다.
- 설명란이 열린 상태에서 `Back to Index`를 눌러도 전환 중 남아 있지 않도록 selector 우선순위를 보정했습니다.
- 전환 마지막에 설명란이 왼쪽으로 한 번 더 움직여 보이던 문제도 숨김/visibility 처리로 정리했습니다.

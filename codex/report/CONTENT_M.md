# M - Mimic 개발 보고서

## 1. 작업 개요

- 대상 파일: `src/animate-js/class/animate/m_mimic.jsx`
- 연동 파일: `src/animate-js/class/main_menu/wheel_button_item.jsx`
- 보고 일자: `2026-04-12`
- 작업 목표: `MIMIC` 한 세트가 살아 있는 슬라임 타이포처럼 반응하고, 닫힌 도형에서는 그 한 세트 자체가 내부를 채우도록 변형

## 2. 피드백 반영 사항

사용자 피드백의 핵심은 명확했다.

- 여러 개의 `MIMIC` 군집을 만들지 말 것
- 닫힌 도형에서도 `MIMIC` 한 세트만 유지할 것
- shape 내부를 채우는 주체가 복제된 텍스트가 아니라, 원래 중앙에 있던 `MIMIC` 자체여야 할 것

이 피드백에 따라 기존의 내부 다중 군집 로직은 제거했다.

추가로 이번 단계에서는 다음 방향을 먼저 반영했다.

- 글자 본체를 `fillText`로 찍지 말 것
- `M`, `I`, `C`를 직접 `Path`로 만들어 렌더링할 것
- 글자 모양을 먼저 다듬기 위해, 슬라임/얼굴/브리지/shape 반응 등 부가 효과를 화면에서 제거할 것

이에 따라 현재 `MIMIC`의 문자 본체는 커스텀 벡터 path 기반으로 그려지고 있다.

## 3. 현재 구현 방향

### 3.1 기본 상태

- 중앙에 하나의 `MIMIC` 워드가 존재
- 각 문자는 눈과 입을 가진 슬라임 생물처럼 반응
- 사용자의 포인터와 드로잉에 따라 시선, 입, 찌그러짐, 늘어짐이 계속 바뀜

### 3.2 열린 선 반응

- 사용자가 닫히지 않은 선을 그리면 `MIMIC` 다섯 글자가 선을 따라 늘어남
- 각 글자는 개별적으로 회전하고 찌그러지며, 선의 흐름을 따라가는 슬라임 타이포처럼 보이도록 처리

### 3.3 닫힌 도형 반응

- 닫힌 도형이 만들어지면 `MIMIC`이 복제되지 않음
- 대신 원래의 다섯 글자가 shape 내부 여러 높이로 퍼지며 크게 눌리고 늘어남
- 글자와 글자 사이는 슬라임 브리지로 이어져, 하나의 워드가 shape 안쪽으로 번져 들어간 것처럼 보이도록 구성
- 렌더링은 shape 내부로 clip 해서, 외곽선을 따라 도는 것이 아니라 내부를 점유하는 감각을 우선

## 4. 구현 상세

### 4.1 다중 군집 로직 제거

이전 버전에서는 닫힌 도형 내부를 여러 개의 작은 `MIMIC`으로 채우는 로직이 있었다.
이번 수정에서는 그 로직을 제거했다.

삭제한 방향:

- 내부 반복 배치용 `MIMIC` 군집 생성
- 도형 내부를 여러 워드로 채우는 스캔라인 텍스트 분산 방식

### 4.2 문자 렌더링의 Path 전환

기존에는 문자 본체를 버퍼 캔버스에 `fillText` / `strokeText`로 만든 뒤 왜곡에 사용했다.
이번 단계에서는 이 구조를 바꿔, `M`, `I`, `C`를 직접 그리는 `Path2D`를 정의하고 그 path를 버퍼 캔버스에 렌더링하도록 수정했다.

즉 현재 글자 본체는 폰트 글리프를 그대로 찍는 방식이 아니라:

- 문자별 비율 정의
- 문자별 커스텀 path
- path 기반 fill / stroke

를 통해 만들어진다.

현재 `M`은 특히 다음 기준으로 다시 조정했다.

- 양쪽 세로획과 가운데 대각획의 두께를 맞춤
- 가운데 대각선 두 개가 중앙에서 뾰족한 `V`로 만나도록 재구성
- 즉 하나의 얇은 골이 아니라, 실제 두께를 가진 두 빗변이 중심을 이루는 구조로 수정
- 또한 분리된 직사각형 조합이 아니라, 외곽선이 한 번에 이어지는 단일 `M` path로 다시 작성

`I` 역시 같은 원칙으로 정리했다.

- 위 막대, 몸통, 아래 막대를 분리 도형으로 그리지 않음
- 하나의 외곽선 path가 전체 글자를 한 바퀴 감싸는 구조로 수정

`C`도 이어서 정리했다.

- 바깥 곡선과 안쪽 곡면을 따로 분리하지 않음
- 오른쪽이 열린 하나의 연속 외곽선 path로 다시 작성
- 이후 형태 안정성을 위해, 자유곡선 기반에서 원형/타원형 링을 잘라낸 구조로 다시 정리

추가로 전체 워드의 기본 인상도 한 단계 조정했다.

- 모든 글자의 width ratio를 키워 전반적으로 더 뚱뚱하게 조정
- `M`과 `I`는 꺾이는 부분에 quadratic curve를 넣어 조금 더 둥글게 보이도록 수정
- `C`는 링 두께를 늘리고 개구부 각도를 줄여 더 통통한 형태로 조정

## 4.5 드로잉 입력 규칙

사용자가 마우스로 경로를 그릴 수 있도록 드로잉 입력을 다시 붙였다.

- 드래그 중에는 자유 경로를 그대로 표시
- 새 선분이 기존 선분과 교차하면, 그 교차 루프를 우선 도형으로 추출
- 추출된 루프는 convex hull로 보정해 오목다각형이 생기지 않도록 처리
- 교차가 없더라도 시작점과 끝점이 충분히 가까우면 닫힌 도형으로 판단하고, 이 경우에도 convex hull 보정을 적용

즉 현재는 “교차로 만들어진 영역을 도형으로 잡고, 결과 도형은 항상 볼록하게 정리하는” 입력 규칙으로 정리된 상태다.

## 4.6 도형 완성 후 타이포 변형

도형이 확정되면 `MIMIC` 글자가 다시 기본 중앙 배치에 머무르지 않고, 계산된 `shapeTargets`에 맞춰 이동하도록 연결했다.

- 닫힌 도형이 확정되면 다섯 글자가 도형 내부 목표 좌표로 이동
- 각 글자는 position뿐 아니라 scale, rotation, shear도 함께 보간
- 따라서 결과적으로 하나의 `MIMIC` 워드가 완성된 도형 구조에 맞춰 찌그러지고 배치되는 상태가 됨

이후 사용자 피드백에 따라 이 방식은 다시 수정했다.

- 글자를 도형 내부 좌표로 “재배치”하는 느낌이 강해 의도와 어긋남
- 현재는 단일 `MIMIC` 워드 전체를 offscreen buffer에 만든 뒤
- 닫힌 도형의 스캔라인 폭에 맞춰 row-by-row로 늘여 그리는 방식으로 변경

즉 지금은 개별 글자가 흩어지는 방식이 아니라, 원래 한 줄로 있던 `MIMIC`이 도형 내부에서 통째로 warp되는 구조다.

### 4.3 효과 제거 상태

글자 형태를 먼저 다듬기 쉽도록, 현재 렌더 단계에서는 다음 요소를 전부 비활성화했다.

- 슬라임 blob
- 눈과 입
- 그림자
- 글자 사이 브리지
- 닫힌 도형 shape fill 연출
- 가이드성 caption 및 보조 연출

지금 화면에서는 중앙의 `MIMIC` 벡터 문자 형태만 확인할 수 있다.

### 4.4 단일 워드 변형 방식

닫힌 도형일 때는 polygon 내부 스캔라인을 사용해 다섯 글자의 목표 위치를 다시 계산한다.
이때 목적은 여러 워드를 만드는 것이 아니라, `MIMIC` 한 세트의 각 글자가 도형 내부에서 최대한 퍼지도록 재배치하는 것이다.

추가로:

- 각 글자의 local span에 따라 scale을 키움
- 글자 사이에 유기적인 slime bridge를 그림
- 전체 렌더링을 shape 내부로 제한해 내부 채움 느낌을 강화

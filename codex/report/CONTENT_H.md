# H - Homebody 개발 보고서

## 1. 작업 개요

- 대상 파일: `src/animate-js/class/animate/h_homebody.jsx`
- 작업 일자: `2026-04-05`
- 테마: `Homebody`
- 목표: `Homebody` 타이포그래피와 집 그래픽을 결합하고, `body` 글자를 집 안에 배치한 뒤 사용자가 각 글자를 드래그로 꺼냈다가 놓으면 황급히 집으로 돌아가게 만들기

## 2. 핵심 구현 방향

기존 버전은 `body` 글자를 드래그할 수 있고, 놓으면 원래 자리로 천천히 직선 복귀하는 구조였습니다.
이번 수정에서는 아래 두 가지를 중심으로 다시 설계했습니다.

- `Home` 텍스트와 집 그래픽을 하나의 장면으로 다시 정리
- `body` 각 글자가 상하좌우만 사용해 복귀 경로를 따라 돌아오도록 변경

단순 보간으로는 "급히 집으로 돌아가는 느낌"이 약하다고 판단해서, 복귀 시에는 경유 지점을 따라 움직이는 구조로 바꿨습니다.

## 3. 구현 내용

### 3.1 장면 레이아웃 재정리

- `Home` 텍스트는 집 왼쪽에 고정 배치
- 집은 본체, 지붕, 굴뚝, 창문, 문으로 구성
- `body`는 집 내부 폭을 기준으로 다시 계산해서 항상 집 안에 들어가도록 정렬
- 캔버스 크기에 따라 `fontSize`, 집 크기, 내부 여백, 경로 격자 간격이 같이 반응하도록 구성

즉, 이번 버전에서는 "타이포그래피 + 아이콘"이 아니라 실제로 `body`가 집의 내부 구성원처럼 보이도록 맞췄습니다.

### 3.2 글자 상태 관리 구조 변경

기존 전역 배열 기반 구조를 상태 객체 중심으로 다시 정리했습니다.

각 글자는 다음 정보를 독립적으로 가집니다.

- 현재 위치
- 원래 위치
- 글자 폭과 높이
- 드래그 중 여부
- 포인터 오프셋
- 복귀 경로 배열
- 현재 이동 중인 경로 인덱스
- 복귀 속도

이 구조 덕분에 `b`, `o`, `d`, `y`가 서로 독립적으로 드래그되고, 각자 다른 경로로 돌아가도 자연스럽게 처리됩니다.

### 3.3 드래그 인터랙션 개선

- 포인터 다운 시 가장 위에 있는 글자를 선택
- 선택된 글자는 배열 뒤로 보내서 항상 최상단에 그려지게 처리
- 드래그 중에는 포인터와 글자 사이의 오프셋을 유지해 손에 붙어 움직이는 느낌을 유지
- 포인터 업 시 즉시 복귀 경로를 계산하고 자동 귀가 시작

기존처럼 단순히 클릭 판정 후 좌표를 더하는 방식보다, 드래그 감각과 레이어 순서가 훨씬 안정적으로 정리되었습니다.

### 3.4 복귀 경로를 직선에서 격자 이동으로 변경

이번 작업의 핵심은 여기입니다.

복귀 시에는 현재 위치에서 집 안의 원래 자리까지 `A*`에 가까운 격자 탐색 방식으로 경로를 만듭니다.
이때 경로는 상하좌우 네 방향만 허용합니다.

경로 계산 시 장애물로 취급한 요소는 아래와 같습니다.

- `Home` 텍스트 영역
- 집 외벽 두께 영역
- 지붕 영역
- 굴뚝 영역

반대로 집 안으로는 무조건 막히지 않도록 하지 않고, 문 쪽 통로(`doorPassageRect`)를 열어 둬서
글자들이 집 바깥에서 안으로 들어올 때 문 근처를 통해 복귀하는 흐름이 나오도록 했습니다.

그래서 결과적으로 글자들은 다음처럼 움직입니다.

- 바깥으로 꺼내면 자유롭게 드래그 가능
- 손을 놓으면 빠르게 경로 계산
- `Home`과 집 외곽을 피함
- 상하좌우만 사용하며 꺾어서 이동
- 마지막에는 집 안 자기 자리로 정렬

### 3.5 복귀 속도와 인상 조정

복귀는 "천천히 끌려온다"보다 "황급히 돌아간다"에 가깝게 보이도록 속도를 점진적으로 올리게 만들었습니다.

- 시작 속도는 낮게
- 프레임마다 가속
- 최대 속도는 제한
- 한 구간이 끝나면 다음 구간으로 바로 전환

이렇게 해서 경로는 꺾이지만, 전체 인상은 머뭇거리지 않고 급하게 집으로 돌아가는 느낌에 가깝게 맞췄습니다.

### 3.6 시각 보조 요소

- 복귀 중인 글자에는 아주 약한 경로 가이드 라인을 추가
- 집 내부에 창문과 문을 넣어 "집 안"이라는 해석이 더 분명해지도록 조정
- `body` 글자는 노란색 톤과 그림자를 적용해 집 안에서 눈에 띄게 처리

가이드 라인은 과하게 튀지 않도록 낮은 투명도로만 사용했습니다.

추가로 더 다듬고 싶다면 다음 방향이 자연스럽습니다.

- 글자별로 복귀 속도나 성격을 조금씩 다르게 주기

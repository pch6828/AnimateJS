# CONTENT F 개발 보고서

## 1. 작업 개요

- 대상 파일: `src/animate-js/class/animate/f_fortune.jsx`
- 연결 파일: `src/animate-js/class/content_page/content_item.jsx`
- 주제: `Fortune`
- 콘셉트: `FORTUNE`의 각 문자가 포춘쿠키가 되고, 터치로 쿠키를 깨서 운세 쪽지를 확인하는 인터랙션
- 작업 일자: `2026-05-10`

## 2. 작업 목표

- `F`, `O`, `R`, `T`, `U`, `N`, `E` 각 글자를 독립적인 포춘쿠키로 표현한다.
- 각 글자 쿠키를 여러 번 터치하면 쿠키가 갈라지고 조각이 아래로 떨어지게 한다.
- 쿠키 안에서 운세가 적힌 쪽지가 나오고, `FORTUNE` 문자 아래 중앙으로 자연스럽게 내려오게 한다.
- 운세 쪽지를 한 번 클릭하면 쪽지가 사라지고, 해당 위치에서 사라졌던 포춘쿠키가 위에서 다시 내려오게 한다.
- `F` 콘텐츠가 실제 페이지에서 접근 가능하도록 콘텐츠 목록에 연결한다.

## 3. 최종 구현 요약

기존 `f_fortune.jsx`는 화면 중앙의 단일 포춘쿠키에서 종이를 끌어내는 구조였다. 이번 작업에서는 이를 `FORTUNE`의 7개 글자 쿠키가 각각 상태를 갖는 구조로 전면 개편했다.

각 쿠키는 다음 흐름으로 동작한다.

1. 기본 상태에서는 `FORTUNE` 글자 모양의 포춘쿠키로 배치된다.
2. 사용자가 같은 쿠키를 3번 터치하면 쿠키에 균열 표현이 쌓인다.
3. 3번째 터치에서 쿠키가 좌우 조각으로 갈라지고, 조각은 중력과 회전을 받으며 아래로 떨어진다.
4. 동시에 운세 쪽지가 쿠키 위치에서 생성되어 화면 아래 중앙 목표 위치로 이동한다.
5. 사용자가 쪽지를 클릭하면 쪽지는 축소 및 페이드아웃되고, 해당 쿠키가 화면 위에서 다시 떨어져 원래 자리로 복귀한다.

## 4. 주요 구현 내용

### 4.1 글자별 쿠키 상태 관리

`WORD = "FORTUNE"`를 기준으로 7개의 쿠키 상태를 생성했다. 각 쿠키는 다음 정보를 갖는다.

- 글자 인덱스와 문자
- 현재 상태: `whole`, `breaking`, `open`, `returning`
- 터치 횟수
- 터치 피드백용 pulse 값
- 운세 메시지
- 쪽지 위치, 속도, 투명도, 크기
- 갈라진 쿠키 조각의 위치, 속도, 회전값

이를 통해 한 글자 쿠키가 깨져 있어도 다른 글자 쿠키는 계속 독립적으로 터치하고 열 수 있다.

### 4.2 터치 인터랙션

마우스 또는 터치 입력은 `movement.isDown`의 상승 에지에서 한 번만 처리한다. 누르고 있는 동안 반복 입력이 발생하지 않도록 `prevIsDown` 기반으로 제어했다.

입력 처리 우선순위는 다음과 같다.

1. 열린 운세 쪽지를 클릭했는지 검사한다.
2. 쪽지를 클릭했다면 해당 쿠키를 `returning` 상태로 전환한다.
3. 쪽지가 아니라면 온전한 쿠키 중 포인터가 닿은 쿠키를 찾는다.
4. 쿠키를 클릭했다면 터치 횟수를 증가시키고, 3회째에 쿠키를 깨뜨린다.

### 4.3 포춘쿠키 파괴 애니메이션

쿠키가 깨질 때 좌우 두 조각을 생성한다. 각 조각에는 서로 다른 방향의 수평 속도, 초기 상향 속도, 중력, 회전 속도를 부여했다.

결과적으로 쿠키가 단순히 사라지는 것이 아니라, 실제로 갈라진 조각이 튕겨 나간 뒤 아래로 떨어지는 느낌을 만든다. 떨어지는 동안 조각의 투명도도 점차 낮아져 화면 밖으로 자연스럽게 사라진다.

### 4.4 운세 쪽지 이동

쪽지는 깨진 쿠키의 위치에서 작게 생성되고, 스프링에 가까운 보간 방식으로 `FORTUNE` 아래 중앙 목표점까지 이동한다.

쪽지 애니메이션에는 다음 요소를 적용했다.

- 크기 증가
- 투명도 증가
- 목표 위치로 이동하는 감쇠 속도
- 초기 회전값이 서서히 정렬되는 움직임

이 방식으로 쪽지가 순간 이동하지 않고, 쿠키 안에서 빠져나와 아래 중앙에 놓이는 흐름으로 보이게 했다.

### 4.5 쪽지 클릭 후 쿠키 복귀

운세 쪽지를 클릭하면 해당 쿠키는 `returning` 상태가 된다.

이때 동작은 다음과 같다.

- 쪽지는 투명도와 크기를 줄이며 사라진다.
- 해당 쿠키의 조각 상태는 제거된다.
- 새 쿠키는 화면 위에서 생성되어 원래 글자 위치로 떨어진다.
- 복귀가 끝나면 상태가 다시 `whole`로 돌아가고, 새로운 운세 메시지가 배정된다.

따라서 같은 글자를 반복해서 열어 보아도 매번 새 쿠키처럼 동작한다.

## 5. 시각 디자인

포춘쿠키의 인상을 살리기 위해 따뜻한 갈색과 노란색 계열을 사용했다.

- 쿠키: 밝은 하이라이트, 기본 갈색, 어두운 가장자리 색을 가진 그라디언트
- 쪽지: 아이보리색 종이, 붉은 가로선, 약한 그림자
- 배경: 따뜻한 종이색과 은은한 곡선 패턴
- 글자: 쿠키 위에 진한 갈색 글자로 표시

각 쿠키는 타원형이 아니라 접힌 포춘쿠키처럼 보이도록 베지어 곡선 기반의 비대칭 형태로 그렸다.

## 6. 콘텐츠 연결

등록된 항목은 다음 정보를 포함한다.

## 9. 결과

`F - Fortune` 콘텐츠는 단일 쿠키를 조작하던 기존 초안에서, `FORTUNE` 전체 단어가 직접 상호작용하는 애니메이션으로 확장되었다. 사용자는 각 글자를 여러 번 터치해 쿠키를 깨고, 운세 쪽지를 읽고, 쪽지를 클릭해 쿠키를 다시 불러오는 순환형 인터랙션을 경험할 수 있다.

## 10. 추가 수정 사항

초기 구현에서는 포춘쿠키 도형 위에 글자를 올린 형태였으나, 최종 콘셉트에 맞춰 글자 자체가 포춘쿠키가 되도록 렌더링 방식을 수정했다.

- 타원형 쿠키 도형을 제거하고 `FORTUNE` 각 글자 glyph를 직접 쿠키 재질로 렌더링한다.
- 글자에는 쿠키색 그라디언트, 진한 가장자리 stroke, 은은한 하이라이트, 표면 질감을 적용했다.
- 쿠키가 깨질 때도 일반 쿠키 조각이 아니라 해당 글자 glyph의 좌우 반쪽이 분리되어 떨어지도록 변경했다.
- 질감과 균열 표현이 글자 바깥 배경에 새지 않도록 오프스크린 캔버스에서 glyph를 먼저 만든 뒤 본 캔버스에 합성한다.

## 11. 조각 파괴 연출 개선

좌우 반쪽으로만 나뉘던 초기 파괴 연출은 너무 인위적으로 보여, 균열과 파편 구조를 다시 조정했다.

- 글자 중앙에 무작위성이 있는 지그재그 균열선을 생성한다.
- 균열선을 기준으로 좌우 2조각이 아니라 위, 중간, 아래 영역을 포함한 여러 비정형 조각으로 나눈다.
- 각 조각은 자신의 중심점을 기준으로 회전하고, 수평 속도, 상승 속도, 회전 속도를 조금씩 다르게 받는다.
- 조각은 글자 전체 이미지를 단순히 자르는 것이 아니라, 각 파편 polygon으로 clipping한 glyph 조각으로 그려진다.
- 조각 외곽에도 어두운 stroke를 추가해 쿠키가 깨진 단면처럼 보이도록 보정했다.

## 12. 균열과 파편 외곽선 수정

피드백에 따라 균열과 파편 렌더링을 추가로 보정했다.

- 터치 중 나타나는 균열을 여러 개의 분리된 짧은 선이 아니라 하나로 이어진 지그재그 경로로 변경했다.
- 터치 횟수가 늘어날수록 같은 균열 경로가 점점 길어지도록 처리해 `>>`처럼 보이던 문제를 제거했다.
- 작은 가지 균열도 메인 균열선의 한 지점에서 시작하도록 만들어 모든 금이 연결되어 보이게 했다.
- 파편 polygon의 외곽 stroke를 제거했다. polygon은 이제 clipping 영역으로만 사용되며, 깨지는 순간 별도의 인위적인 외곽선이 나타나지 않는다.

## 13. 운세 교체 이벤트 수정

운세 쪽지를 닫지 않은 상태에서 다른 쿠키를 깨는 경우의 이벤트 흐름을 수정했다.

- 새 쿠키가 3번째 터치로 깨지는 순간, 이미 열려 있던 다른 쿠키들은 자동으로 `returning` 상태가 된다.
- 이전 쿠키는 사용자가 쪽지를 직접 클릭하지 않아도 위에서 다시 내려와 복귀한다.
- 이전 쪽지는 쿠키 위치로 빨려 들어가며 사라지는 대신, 아래 방향으로 떨어지면서 사라진다.
- 쪽지 낙하에는 좌우 흔들림, 약한 회전, 느린 페이드아웃을 적용해 살랑거리며 떨어지는 느낌을 만들었다.

## 14. 원형 배치와 중앙 확대 이벤트

쿠키 배치를 직선형 단어 구조에서 원형 배치 구조로 변경했다.

- 기본 상태에서는 `FORTUNE`의 각 글자 쿠키가 화면 중앙을 기준으로 한 원 둘레에 배치된다.
- 사용자가 쿠키를 한 번 터치하면 해당 쿠키가 원 둘레 위치에서 중앙으로 이동한다.
- 중앙으로 이동하는 동안 `focusProgress`를 기반으로 위치와 크기를 부드럽게 보간한다.
- 선택된 쿠키는 다른 쿠키보다 나중에 그려 중앙 확대 상태에서 시각적으로 위에 올라오게 했다.
- 중앙에 온 쿠키는 확대된 상태로 추가 터치를 받아 균열이 생기고, 3번째 터치에서 확대된 크기 기준으로 깨진다.

## 구현 내용
### 1. 애니메이션 구조 재정리
- 기존의 컵, 물결, 낙하 줄기 로직을 상태 기반 구조로 다시 정리했다.
- `createState`, `getMetrics`, `updateState`로 상태 생성, 반응형 치수 계산, 프레임별 갱신 책임을 분리했다.
- 배경, 받침, 컵 실루엣, 커피 채움, 윤곽선, 김, 낙하 줄기를 각각 별도 함수로 나눠 이후 수정이 쉬운 형태로 정리했다.

### 2. 머그잔처럼 보이는 COFFEE 텍스트
- `COFFEE` 텍스트 자체를 컵 몸통처럼 보이도록 사용하고, 하단 반원과 손잡이를 추가해 머그잔 실루엣을 만들었다.
- 컵 본체는 크림색, 외곽선은 진한 브라운 톤으로 구성해 형태가 더 잘 읽히도록 조정했다.
- 컵 아래에는 받침 접시와 그림자를 추가해 정적인 포스터 같은 안정감을 더했다.

### 3. 커피가 따라지고 차오르는 인터랙션
- 마우스를 누르고 있는 동안만 위에서 커피 줄기가 내려오도록 수정했다.
- 커피는 즉시 꽉 차지 않고 `targetLevel`과 실제 수위 `level`을 분리해 천천히 차오르도록 만들었다.
- 줄기가 닿을 때마다 `ripple` 값을 올려 수면이 물결치도록 구성했다.

### 4. 수위 감소와 감쇠
- 입력이 멈춘 직후 바로 줄지 않고, 일정 시간 이후 서서히 커피 수위가 낮아지도록 했다.
- 오래 비워두면 조금 더 빠르게 줄어들도록 감쇠 속도를 두 단계로 나눴다.

### 5. 레트로 무드 강화
- 청록 계열 배경 그라디언트와 원형 광원을 넣어 복고풍 카페 사인 느낌을 만들었다.
- 배경 가로 스트라이프를 추가했고, 이후 피드백을 반영해 더 이상 움직이지 않도록 고정했다.
- 커피가 어느 정도 차면 은은한 김이 올라오도록 해 장면이 너무 정적으로 보이지 않게 했다.

### 6. 콘텐츠 메타데이터 보강
- `items['C']`의 `backgroundColor`를 새 화면 톤에 맞게 조정했다.
- 비어 있던 `text`, `toolTipText`를 현재 애니메이션 의도에 맞게 채웠다.

## 후속 수정 이력
### 1차 후속 수정
- 커피가 컵 밖으로 삐져나와 보이던 문제를 줄이기 위해 채움 영역을 다시 제한했다.
- 위에서 내려오는 커피가 화면 중간에서 시작하지 않도록 상단 시작 형태로 수정했다.
- 컵 안쪽으로 들어가는 줄기를 추가해 “잔 안으로 들어가는 흐름”이 보이도록 보완했다.
- 배경 스트라이프는 고정 표현으로 변경했다.

### 2차 후속 수정
- 커피 채움 경로를 사각형 기준에서 컵 하단 반원 기준으로 바꿨다.
- 액체가 직사각형처럼 쌓이는 인상을 줄이고, 컵 아웃라인을 따라 차오르도록 조정했다.

### 3차 후속 수정
- 수위가 반원 중심보다 낮을 때 반원 전체를 닫아버리면서 바깥쪽에 채움이 생기던 문제를 수정했다.
- 현재 수면과 반원이 만나는 지점을 계산해, 잠긴 구간의 호만 따라 채우도록 변경했다.
- 이 수정으로 낮은 수위에서 반원 바깥으로 번지던 채움 문제를 보정했다.

### 4차 후속 수정
- 위에서 따라지는 커피 줄기가 컵 입구에서 끊겨 보이던 문제를 줄이기 위해 바깥 줄기와 안쪽 줄기의 연결 구간을 다시 다듬었다.
- 접점이 형상보다 색 차이 때문에 더 도드라진다는 피드백을 반영해, 안쪽 줄기와 바깥 줄기가 같은 메인 컬러와 같은 하이라이트 톤을 사용하도록 통일했다.
- 메인 스트림을 먼저 바깥쪽과 안쪽까지 모두 그린 뒤, 그 위에 하이라이트를 전체로 이어서 그리도록 렌더링 순서를 재구성했다.
- 마지막으로 안쪽과 바깥쪽 스트림의 굵기도 동일한 기준으로 맞춰, 컵 입구에서 두께 변화 때문에 접점이 튀어 보이지 않도록 정리했다.
